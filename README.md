# Customer Analysis

О проекте

Цель проекта — рассчитать коэффициенты пролонгации по каждому менеджеру и по всему отделу за год.

Проект выполнен в формате аналитического исследования с использованием Python, pandas и Jupyter Notebook.



**Важно!!**

Исходные CSV-файлы были получены из внешнего источника и содержали проблемы с кодировкой, поэтому не прикладываются к проекту.
Для корректной обработки данных использовалась locale.

**Цель проекта**

Руководителю отдела было важно понять:

- насколько успешно менеджеры продлевают клиентов;
- сколько проектов продлеваются сразу после завершения;
- сколько клиентов возвращаются позже;
- какие менеджеры показывают лучшие результаты.


Что было сделано

**В рамках проекта:**

- очищены и обработаны данные;
- обработаны специальные значения в таблицах;
- рассчитаны коэффициенты пролонгации;
- выполнен анализ по менеджерам и месяцам;
- построены аналитические таблицы и графики;
- подготовлен итоговый отчет.

**В проекте использовались две таблицы:**

*prolongations.csv*

Таблица содержит:

-id проекта;
-месяц завершения проекта;
-аккаунт-менеджера.

*financial_data.csv*

Таблица содержит:

-id проекта;
-финансовые данные по месяцам;
-причины дублей;
-ответственного менеджера.

**Особенности данных**

В данных встречались специальные значения:

Значение:
- 'в ноль'	- отгрузка в этом месяце равна 0
- 'stop' -	проект завершён досрочно
- 'end'	- проект завершён

*Обработка значения в ноль*

Если в месяце завершения проекта встречалось значение в ноль, это означало, что фактическая отгрузка была равна 0.

По условию задачи в таком случае для расчета коэффициента пролонгации нужно было использовать отгрузку предыдущего месяца.

Проекты со значениями stop и end исключались из анализа.

**Что рассчитывалось**

*Коэффициент пролонгации в первый месяц* показывает, сколько проектов были продлены сразу после завершения.

Для расчета использовалось отношение:
суммы проектов, продленных в следующий месяц к сумме всех завершившихся проектов.

*Коэффициент пролонгации во второй месяц* показывает проекты, которые были продлены с задержкой.

Для расчета использовалось отношение: 
суммы проектов, продленных через месяц к сумме проектов, не продленных сразу.
